# 48-Hour Technical Interview Problems

This repository is a portfolio entry based on a technical interview round I went through.

I had 48 hours to solve the assigned problems. The aim here is not only to show the final answers, but also to make the route to them visible: how I read the tasks, separated the hard parts from the mechanical ones, tested assumptions, and decided what was good enough under a fixed deadline.

It is meant to record technical judgement under pressure.

## What is here

Depending on the problem, the repository includes:

- solution code;
- short explanations of the approach;
- assumptions and constraints;
- tests or validation steps;
- notes on edge cases;
- things I would improve with more time.

## How to read it

Start with the problem summaries and approach notes. Then look at the implementation and tests.

The main value of the repository is in the combination of code and reasoning. It is meant to show how I work when the problem is demanding, the requirements are not fully clean, and the time is limited.

## Confidentiality

The original interview details have been removed, anonymized, or paraphrased. What remains is my own work and my own account of the reasoning behind it.
